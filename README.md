
# Num Grid

**当前版本：v0.2.0**

[![Rust](https://img.shields.io/badge/rust-1.70%2B-blue.svg)](https://www.rust-lang.org/)
[![License](https://img.shields.io/badge/license-MIT%2FApache--2.0-green.svg)](LICENSE)

一个探索数字填充游戏最优解的高性能并行搜索算法。

## 游戏规则

1. 在无限网格的指定坐标上填入数字 `1`（可自定义位置）。
2. 从 `2` 开始，按顺序将下一个自然数填入一个**边界空格**。
3. **边界空格**：至少与一个已填数字相邻（八方向）。
4. **可填条件**：该空格周围所有已填数字的**非空子集和**中包含当前要填的数字。
5. 无法填入时游戏结束。目标：使最后填入的数字尽可能大。

## 示例

初始 `1` 位于 `(0,0), (1,0)` 时，最优解可达较高数字（具体视搜索参数）。运行程序将输出最终网格和最大数字。

## 编译与运行

### 环境要求

- Rust 1.70 或更高版本
- Cargo

### 获取代码

```bash
git clone https://github.com/user-lcd/num-grid.git
cd num-grid
```

### 编译

```bash
cargo build --release
```

### 运行

```bash
cargo run --release
```

程序会自动开始搜索，并在找到更优解时打印当前网格和最大数字。最终输出最终最优网格。

## 配置

在 `main.rs` 中可以修改以下常量：

- `INIT_POSITIONS`：初始填入 `1` 的坐标数组。例如 `&[(0,0), (1,0)]`。
- `PARALLEL_THRESHOLD`：并行阈值，默认为 `4`。可根据 CPU 核心数和状态分支因子调整。

## 算法亮点

- **对称性剪枝**：自动识别初始布局的对称性，仅生成不等价的初始状态。
- **自适应并行 DFS**：基于后继数量动态切换串行/并行模式，充分利用多核。
- **高效边界表示**：预计算邻居子集和，后继生成 O(1) 判断可填性。
- **原子全局最优**：线程安全地记录当前最优解，无需全局锁竞争。
